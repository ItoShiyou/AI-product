# オセロ（GUI/CUI 分離版）

この実装は、ゲームロジックと表示処理を分離しています。

- ゲームロジック: 盤面状態の更新、着手可否判定、石の反転、終了判定
- GUI: Canvas 描画とマウス入力
- CUI: 文字表示とキーボード入力

GUI と CUI は同じロジック関数を利用するため、挙動が一致します。

## 起動方法

### ブラウザ版

1. index.html をブラウザで開く
2. 盤面をクリックして着手する
3. リセットボタンで初期状態に戻す

### CUI 版

1. ターミナルでこのフォルダに移動
2. node main.js を実行
3. 入力形式は x y（例: 2 3）
4. q で終了

表示記号:

- B: 黒石
- W: 白石
- *: 着手可能マス
- .: 空きマス

## 実装の構成

- 状態変更はロジック関数群に集約
- GUI は描画とクリック入力のみ
- CUI は文字描画と入力ループのみ
- 起動時に実行環境を判定し、GUI か CUI を選択

## 関数一覧（トップレベル）

### createInitialBoard

初期配置の 8x8 盤面を作成します。

### createGameState

ゲーム状態オブジェクトを初期化します。

- board: 盤面
- turn: 現在手番
- gameOver: 終了フラグ
- winner: 勝者（引き分け時は null）
- passCount: 連続パス回数

### opponent

指定した色の相手色を返します。

### isInsideBoard

指定座標が盤面内かどうかを返します。

### countStones

黒石数と白石数を数えて返します。

### findFlippableStones

指定位置に着手したときに裏返る石の座標一覧を返します。

### isPuttable

指定位置に着手可能かどうかを返します。

### getPuttableMoves

現在手番（または指定色）の着手可能手をすべて返します。

### updateGameOver

次の条件でゲーム終了を更新します。

- 盤面が埋まった
- 黒石または白石が 0
- 連続パスが 2 回

### advanceTurn

手番を進めます。現在手番が打てない場合はパス処理を行い、両者打てない場合は終了判定します。

### applyMove

1 手分の状態変更を行う中核関数です。

- 着手可否判定
- 石を置く
- 反転処理
- 手番更新
- パス処理
- 終了判定

### resetGame

状態を初期状態に戻します。

### colorLabel

色定数を表示用文字列に変換します。

### winnerLabel

勝者情報を表示用文字列に変換します。

### createBrowserApp

ブラウザ UI を構築します。Canvas 作成、描画、クリック処理、リセット処理を含みます。

### formatBoardForCli

盤面を CUI 表示用の文字列に整形します。

### runCli

CUI 実行ループを開始します。描画、入力受付、バリデーション、終了処理を行います。

## createBrowserApp 内部関数

### drawStone

石 1 個を描画します。

### drawBoard

盤面背景、格子、既存石を描画します。

### drawPuttable

着手可能マスにマーカーを描画します。

### drawScores

黒白スコア表示を更新します。

### render

画面描画の入口です。盤面、着手候補、スコアを順に描画します。

### handleEndIfNeeded

ゲーム終了時に結果アラートを表示します。

### tryMove

クリック座標をロジックへ渡し、着手成功時のみ再描画します。

### reset

状態をリセットし、再描画します。

## runCli 内部関数

### renderCli

CUI の盤面、手番、スコア、入力ガイドを表示します。

### finishCli

最終スコアと勝者を表示して CUI を終了します。

### askMove

入力ループ本体です。

- 入力受付
- 入力形式チェック
- 座標チェック
- 着手実行
- 次ターンへ遷移
