
# Project Directive: Minimalist Online Texas Hold'em

## 1. Project Vision
- **Concept:** A clean, minimalist online poker app for mobile and web.
- **Key Experience:** Low-latency, stress-free UI, and fair play via serverless security.
- **Design Philosophy:** Minimalist aesthetics using pastel color palettes, generous white space, and a unified UI across platforms.

## 2. Technical Stack
- **Frontend:** Vanilla JavaScript (ES6+), HTML5 Canvas/CSS3.
- **Native Bridge:** Capacitor (for iOS/Android deployment).
- **Backend (Firebase):**
    - **Realtime Database:** Live turn synchronization and room state.
    - **Cloud Functions:** Secure logic (shuffling, dealing, winner calculation).
    - **Authentication:** Anonymous and Social Auth.
    - **Hosting:** Web version deployment.
- **Monetization:** Google AdMob (Rewarded Video Ads for chip recovery).

## 3. System Architecture & Security
- **Anti-Cheat Logic:**
    - Game state and private hands are managed by Cloud Functions.
    - Players only have read access to their own "private" hand path.
    - Winner determination and chip movement are executed server-side.
- **Resilience:**
    - **Auto-Action:** 20s timeout logic to handle disconnections.
    - **State Recovery:** Clients must resume their specific `roomID` upon reconnection.

## 4. Monetization & Economy
- **Rewarded Ads:** "Bankruptcy Protection" – Watch an ad to refill chips when balance < threshold.
- **The Rake:** 3% system fee per pot to control inflation and encourage ad engagement.
- **Customization:** Achievements and ad-watches unlock minimalist card/table themes.

## 5. Data Schema (Firebase Realtime DB)
```json
{
  "rooms": {
    "roomId": {
      "meta": { "blind": 10, "status": "waiting|playing", "dealer": 0 },
      "public": { "communityCards": [], "pot": 0, "turnIndex": 0 },
      "players": {
        "uid": { "name": "str", "chips": 0, "bet": 0, "state": "acting|folded" }
      }
    }
  },
  "private": {
    "roomId": {
      "uid": ["card1", "card2"] 
    }
  }
}
```

## 6. Implementation Phases
### Phase 1: Core Logic (Prototype)
- JavaScript Hand Evaluator (7-card to 5-card best hand).
- Realtime chip/turn synchronization logic between two clients.
- Minimalist UI prototype (Local vs. simple AI).

### Phase 2: Security & Functions
- Shuffling and dealing logic in Cloud Functions.
- Room management (Create/Join/Auto-Matchmaking).
- Server-side validation of player actions (Check/Call/Raise).

### Phase 3: Polish & Distribution
- AdMob integration for Rewarded Video.
- Capacitor setup for iOS/Android builds.
- UI refinement with pastel themes and smooth CSS animations.
